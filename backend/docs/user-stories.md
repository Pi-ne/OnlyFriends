> **文档版本**：US-1.1  
> **依据文档**：[趣聚平台系统架构与实现方案 v3.0](../趣聚平台系统架构与实现方案_v3.md)  
> **状态**：已定稿

---

## 1. 文档说明

| 项 | 说明 |
|---|---|
| 产品名称 | OnlyFriends（趣聚平台） |
| 层级结构 | **Epic**（史诗）→ **Story**（用户故事） |
| 故事格式 | 作为 \<用户角色\> / 我想要 \<结果\> / 以便于 \<目的\> |
| 优先级 | P0（MVP 必须）/ P1（首发完整版）/ P2（增强） |
| Epic 编号 | `EPIC-{序号}` |
| Story 编号 | `US-{域}-{序号}` |

### 故事书写模板

```text
作为 <用户角色>
我想要 <结果>
以便于 <目的>
```

---

## 2. 角色定义（Personas）

| 角色 ID | 角色名称 | 说明 |
|---------|----------|------|
| R01 | 访客 | 未登录用户，可浏览公开活动 |
| R02 | 个人用户 | 已注册激活的普通用户 |
| R03 | 商家用户 | 通过商家认证的用户，可发起商业类活动 |
| R04 | 活动发起人 | 创建并运营活动的用户（含个人/商家） |
| R05 | 活动参与者 | 报名、签到、评价活动的用户 |
| R06 | 小队队长 | 创建并管理兴趣小队的用户 |
| R07 | 小队管理员 | 被队长授权的小队管理人员 |
| R08 | 小队成员 | 加入小队的普通成员 |
| R09 | 平台管理员 | Web 后台运营人员 |
| R10 | 开发/运维人员 | 负责平台部署与运维的技术人员 |
