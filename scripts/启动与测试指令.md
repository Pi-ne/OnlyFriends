# OnlyFriends 启动与测试指令

本文档中的命令默认都在项目根目录执行。

如果你现在在 `backend` 目录里，先回到项目根目录：

```powershell
cd ..
```

项目根目录完整路径：

```powershell
cd "C:\Users\liuke\Desktop\大三下\软件过程与质量\Onlyfriends\OnlyFriends"
```

## 1. 首次启动前准备

确认本机已安装：

- Docker Desktop
- JDK 17+
- Maven
- Node.js

如果 PowerShell 阻止脚本执行，先执行一次：

```powershell
Set-ExecutionPolicy -Scope CurrentUser RemoteSigned
```

## 2. 启动基础设施

启动 MySQL 和 Redis：

```powershell
cd backend
docker compose up -d mysql redis
cd ..
```

## 3. 初始化数据库

首次运行，或数据库被重建后执行：

```powershell
cd backend
Get-Content .\sql\init-all.sql -Encoding UTF8 | docker exec -i ququ-mysql mysql -uroot -pququ_root_password --default-character-set=utf8mb4
cd ..
```

如果需要导入演示数据：

```powershell
cd backend
Get-Content .\sql\demo-data.sql -Encoding UTF8 | docker exec -i ququ-mysql mysql -uroot -pququ_root_password --default-character-set=utf8mb4
cd ..
```

## 4. 启动后端

推荐后台启动，不会弹出一堆服务窗口：

```powershell
.\scripts\start-all.ps1 -Background
```

后台启动并额外启动 AI 服务：

```powershell
.\scripts\start-all.ps1 -WithAi -Background
```

后台启动日志位置：

```text
backend\logs
```

如果你想每个服务单独开一个窗口，方便实时看日志：

```powershell
.\scripts\start-all.ps1
```

每个服务单独开窗口，并额外启动 AI 服务：

```powershell
.\scripts\start-all.ps1 -WithAi
```

该脚本会启动：

- user-service: `http://localhost:8081`
- activity-service: `http://localhost:8082`
- social-service: `http://localhost:8083`
- im-service: `http://localhost:8084`
- admin-service: `http://localhost:8085`
- gateway: `http://localhost:8080`
- ai-service: `http://localhost:8001`，仅 `-WithAi` 时启动

统一后端入口：

```text
http://localhost:8080
```

## 5. 启动前端测试台

另开一个 PowerShell 窗口，在项目根目录执行：

```powershell
.\scripts\start-frontend.ps1
```

如果也想让前端在后台运行：

```powershell
.\scripts\start-frontend.ps1 -Background
```

如果提示 `address already in use 127.0.0.1:5173`，说明前端已经启动了，直接打开测试地址即可。若需要重启前端：

```powershell
.\scripts\start-frontend.ps1 -Restart
```

前端测试台地址：

```text
http://127.0.0.1:5173
```

## 6. 前端测试流程

打开：

```text
http://127.0.0.1:5173
```

推荐测试顺序：

1. 在“认证与账户”模块注册用户。
2. 激活账号。激活 token 可从 user-service 日志或数据库中获取。
3. 在“认证与账户”模块登录用户。
4. 登录成功后，用户 Token 会自动保存到页面顶部。
5. 依次测试“活动全流程”“AI 能力”“关注与好友”“兴趣小队”“即时通讯 REST”。
6. 在“后台管理”模块登录管理员。
7. 管理员登录成功后，Admin Token 会自动保存，再测试后台审核、封禁、下架、恢复等接口。

默认管理员账号：

```text
username: admin
password: Admin123456
```

## 8. 常用检查命令

查看 Docker 容器：

```powershell
docker ps
```

检查网关是否可访问：

```powershell
Invoke-WebRequest http://localhost:8080 -UseBasicParsing
```

检查前端是否可访问：

```powershell
Invoke-WebRequest http://127.0.0.1:5173 -UseBasicParsing
```

重新编译后端：

```powershell
cd backend
mvn -DskipTests package
cd ..
```

运行后端测试：

```powershell
cd backend
mvn test
cd ..
```

## 9. 关闭服务

如果使用 `-Background` 启动，可以一条命令关闭后端和前端：

```powershell
.\scripts\stop-all.ps1
```

如果前端是在当前窗口启动的：

```text
在运行 node server.js 的窗口按 Ctrl + C
```

如果后端是每个服务单独开窗口启动的：

```text
关闭 start-all.ps1 打开的各个 PowerShell 服务窗口
```

关闭 MySQL 和 Redis：

```powershell
cd backend
docker compose down
cd ..
```
