# OnlyFriends 用户故事

> **文档版本**：US-2.0  
> **依据文档**：OnlyFriends 平台需求 v2、[版本发布计划](./release-plan.md)、[验收标准](./acceptance-criteria.md)  

---

## 1. 文档说明

| 项 | 说明 |
|---|---|
| 产品名称 | OnlyFriends |
| 层级结构 | **Epic**（史诗）→ **Story**（用户故事） |
| 故事格式 | 作为 `<用户角色>` / 我想要 `<结果>` / 以便于 `<目的>` |
| 优先级 | P0（本次实践主链路必须）/ P1（需求要求但可轻量或择优实现）/ P2（增强，可延期） |
| 10 天处理策略 | 第一次迭代必做 / 第二次迭代必做 / 第二次迭代择优 / 实践后延期 |
| Epic 编号 | `EPIC-{序号}` |
| Story 编号 | `US-{域}-{序号}` |

> 本文档在原用户故事基础上，补充需求文档中的硬性要求，并结合 10 天实践周期对故事进行拆分和收敛。文档既保留完整产品蓝图，也明确本次实践的交付优先级。

### 故事书写模板

```text
作为 <用户角色>
我想要 <结果>
以便于 <目的>
```

### 10 天交付规则

| 策略 | 含义 |
|------|------|
| 第一次迭代必做 | 第 3-5 天完成，支撑第一次产品评审和主链路 Demo |
| 第二次迭代必做 | 第 6-9 天完成，支撑互评和最终答辩，通常为需求中的硬性展示点 |
| 第二次迭代择优 | 主链路稳定后补充，若时间不足需记录延期原因 |
| 实践后延期 | 不影响 10 天验收主链路，进入 v1.1+ 后续增强 |

## 2. 角色定义（Personas）

| 角色 ID | 角色名称 | 说明 |
|---------|----------|------|
| R01 | 访客 | 未登录用户，可浏览公开活动和进入登录注册流程 |
| R02 | 个人用户 | 已注册激活的普通用户，可浏览、报名和发起非商业活动 |
| R03 | 商家用户 | 通过商家认证的用户，可发起商业类活动并维护商家资料 |
| R04 | 活动发起人 | 创建、提交、运营活动的用户（含个人/商家） |
| R05 | 活动参与者 | 报名、签到、评价活动的用户 |
| R06 | 小队队长 | 创建并管理兴趣小队的用户 |
| R07 | 小队管理员 | 被队长授权的小队管理人员 |
| R08 | 小队成员 | 加入小队的普通成员 |
| R09 | 平台管理员 | Web 后台运营人员，负责审核与治理 |
| R10 | 开发/运维人员 | 负责平台部署、运行、接口与质量保障的技术人员 |
