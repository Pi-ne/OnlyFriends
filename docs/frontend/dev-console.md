# OnlyFriends 开发管理台使用说明

该页面是给开发和测试使用的 Web 管理台，入口为：

```text
http://127.0.0.1:5173
```

源码文件：

```text
frontend/index.html
frontend/app.js
frontend/styles.css
frontend/server.js
```

## 启动方式

先启动后端：

```powershell
.\scripts\start-all.ps1 -Background
```

测试 AI 独立服务时可加 `-WithAi`。

再启动前端静态服务：

```powershell
.\scripts\start-frontend.ps1
```

打开：

```text
http://127.0.0.1:5173
```

默认 API Base：

```text
http://localhost:8080/api/v1
```

## 默认管理员

```text
username: admin
password: Admin123456
```

登录成功后，管理台会把管理员 Token 缓存在浏览器 localStorage 中。

## 导航模块

管理台左侧共 **7 个功能视图**（`frontend/index.html`）：

| 视图 | 说明 |
|------|------|
| 概览 | 快速检查网关、管理员登录与核心列表接口 |
| 用户管理 | 用户查询、封禁、解封、详情 |
| 活动管理 | 活动查询、审核通过/驳回/要求修改、下架、恢复 |
| 兴趣组/小队 | 小队查询、停用/恢复、成员查看；**双用户联调**（用户 A/B 登录、浏览/加入小队、小队群聊） |
| 好友与聊天 | **双用户联调**：好友申请、好友列表、会话列表、私聊收发、已读标记 |
| 测试造数 | 注册测试用户、普通用户登录、创建活动、创建小队 |
| 接口调试 | 任意网关 API 原始请求与响应日志 |

## 已支持功能

### 平台治理（管理员 Token）

- 用户查询、封禁、解封、详情查看
- 活动查询、审核通过、审核驳回、要求修改、下架、恢复
- 小队查询、停用、恢复、成员查看

### 测试造数（普通用户 Token）

- 创建测试用户（邮箱注册）
- 普通用户登录并缓存 `accessToken` / `userId`
- 使用用户 Token 创建测试活动、创建兴趣组/小队

### 社群与 IM 联调（双用户槽位 A/B）

- 两个独立用户槽位登录，切换当前操作用户
- 好友申请发起、申请列表、同意/拒绝
- 好友列表、私聊会话列表、历史消息拉取、发送文字消息、标记已读
- 小队浏览与加入、已加入小队列表、小队群聊消息收发（WebSocket 经网关 `/ws/im`）

### 通用

- API Base 可配置并持久化到 `localStorage`
- 原始接口调试（自定义 Method / Path / Body，可选管理员或用户 Token）

## 当前后端未提供的能力

管理台不会伪造后端不存在的功能。当前后端公开管理接口中暂未提供：

- 删除用户
- 重置普通用户密码
- 直接编辑任意用户资料

这些能力如果后续需要，可以先在后端补充对应接口，再在 `frontend/app.js` 中接入按钮。

## 使用建议

1. 先登录管理员，在「概览」确认网关与核心列表接口正常。
2. 在「测试造数」注册并登录普通用户，获得用户 Token。
3. 用用户 Token 创建活动或兴趣组，回到「活动管理」完成审核流转。
4. 需要验证好友/私聊时：在「好友与聊天」用用户 A/B 双槽位登录，发起好友申请并私聊。
5. 需要验证小队群聊时：在「兴趣组/小队」双用户加入同一小队，使用小队群聊面板收发消息。
6. 「接口调试」用于排查非 UI 封装的边缘 API。
